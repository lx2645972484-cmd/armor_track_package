# 🎯 RoboMaster 装甲板自瞄系统

## 📌 项目简介
本项目是为 RoboMaster 机器人对抗赛设计的一套基于 ROS 2 的装甲板视觉自瞄系统。系统涵盖了从图像采集、预处理、目标检测、数字识别到三维状态估计和云台控制的完整业务流。项目采用 C++ 与 OpenCV 编写核心视觉逻辑，并结合 PyTorch 训练的轻量级神经网络进行数字分类，最后通过串口将解算后的偏航角 (Yaw) 和俯仰角 (Pitch) 发送至底层电控。

## ✨ 核心功能特性 (Features)

### 1. 稳定平滑的追踪与全链路坐标转换
* **TF2 坐标树集成**：配置了完整的 `urdf` 模型，实现了从 `camera_link` 到云台 `base_link` 的层层空间坐标系转换。
* **时间戳对齐优化**：在进行 TF 查询时优先使用 `rclcpp::Time(0)` 获取最新变换，有效避免了 TF 树中的外推时间报错 (Extrapolation Error)。
* **全双工串口通信**：构建了独立稳定的串口驱动节点，带有 CRC16 校验机制，保障与下位机通信的可靠性。

### 2. 抗跳变的多目标识别与数字分类
* **多目标滞回锁定机制**：设计了基于历史特征和空间距离的目标调度逻辑（Hysteresis），并赋予 "1号英雄" 以及当前锁定目标极高的优先级（特权机制），在相机视野内出现多个目标时能稳定锁敌，不会发生目标频繁跳变。
* **轻量级数字分类网络**：使用自训练的轻量级卷积神经网络（包含 Zenet / LeNet5 架构），通过 PyTorch 训练后导出为 `Zenet-sim.onnx` 格式进行 C++ 前向推理，数字识别准确率极高且推理延迟极低。

### 3. 基于卡尔曼滤波 (KF) 的最优状态估计
* 引入了基于匀速模型 (CV) 的标准卡尔曼滤波器，对单块装甲板的三维空间坐标 `(x, y, z)` 进行平滑处理与超前预测，极大缓解了云台跟随时的延迟和抖动。

### 4. 反 "小陀螺" 旋转中心解算 (EKF)
* **旋转状态建模**：针对敌方机器人的自旋（小陀螺）战术，引入了扩展卡尔曼滤波器 (Extended Kalman Filter, EKF)。通过对装甲板的法向量和空间位置进行观测，推算并维护敌方机器人的中心坐标和旋转半径。
* **🚧 待办事项 (To-Do)**：目前 EKF 已能实现旋转中心的解算，基于此拓展的**整车状态识别与四面装甲板相位对齐功能正在持续开发中**。

### 5. 工业相机深度集成与动态调参
* **大恒 Galaxy 相机驱动**：摒弃了普通的 UVC 驱动，深度封装了大恒工业相机 GxIAPI，支持 Bayer 格式硬件级转换与断线自动重连。
* **相机内参自动加载**：支持从标准的 `.yaml` 文件中一键读取相机内参矩阵 (K) 和畸变系数 (D)，无需硬编码。
* **动态参数调节 (Dynamic Reconfigure)**：在不重启节点的情况下，可以通过 `rqt` 实时调整：
    * 相机曝光时间 (Exposure Time)
    * 模拟增益 (Gain)
    * Gamma 值调整
    * 敌方阵营颜色设定 (0=蓝, 2=红) 以及对应的二值化阈值。

## 🛠️ 系统架构与依赖 (Dependencies)
* **OS**: Ubuntu 22.04 (推荐)
* **ROS 2**: Humble
* **Vision & Math**: OpenCV 4.x, Eigen3
* **Deep Learning**: ONNX Runtime
* **Hardware SDK**: 大恒 Galaxy SDK (GxIAPI)

## 🚀 节点拓扑
* `/armor_tracker`：主视觉节点，负责图像获取、目标检测、解算和滤波预测。
* `/serial_driver`：串口通信节点，负责与下位机进行数据交互。
* `/tf2_ros`：维护 `base_link` -> `turret_base_link` -> `turret_top_link` -> `camera_link` 的空间关系。
